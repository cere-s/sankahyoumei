# トップページ・デザイン刷新 設計書

## 背景

参考イメージ（コスプレ参加表明サイト「コスいく」のランディングページ案）をもとに、トップページの見た目を刷新したい。
現状サイトはすでに同じ世界観（ピンク→バイオレットのグラデーション、「コスいく」ロゴ、キャッチコピー「次のイベント、誰が来るか先に見える。」）を持っており、ゼロからの作り直しではなく、レイアウトの作り込みで参考イメージに近づける。

技術構成は Next.js 16（App Router, RSC）＋ Tailwind CSS v4。新しいライブラリは導入しない。

## 前提・方針（確定事項）

参考イメージには外部素材やデータ拡張を要する要素があるが、今回は次の方針で「素材・データ拡張なし」で成立するデザインに落とす。

- **ヒーローのイラストは使わない。** 2人のコスプレイヤー絵は用いず、グラデーション背景＋テキスト中心のヒーローに設計し直す。吹き出しカード／グラデ帯はCSSで表現する。
- **イベントカードは写真なし。** `Event` 型に画像フィールドは追加しない。日付バッジ＋テキスト（イベント名・会場・参加人数）のカードにする。
- **ランキングは現状維持（イベント別）。** 参考イメージの人物別ランキングには寄せない。人物別はデータ／仕様変更が必要なため対象外。アバターも使わない。

上記により、本設計はすべて **コード変更のみ** で完結する。DBスキーマ変更・画像素材の用意は不要。

## 参考イメージとの対応表

| 参考イメージの要素 | 今回の方針 | 実装区分 |
|---|---|---|
| ヘッダー拡張（イベントを探す／参加表明を探す／ランキング／使い方／お知らせ） | ナビ項目を整理して追加。未作成ページは扱いを決める（後述） | 変更 |
| ヒーロー大見出し＋2CTA | 既存を踏襲しつつ配置強化。イラストは置かない | 微調整 |
| ヒーローの吹き出しカード＋グラデ帯 | CSSで再現 | 新規 |
| 統計カード2枚（開催予定 / 参加表明） | データ既存。カードUI化 | 新規UI |
| 検索・絞り込みバー（役割／イベント日／エリア／作品ジャンル） | `/search` の機能をトップに要約配置 | 移植 |
| 新着の参加表明（横スクロール） | 既存の縦グリッドを横スクロールカルーセルに | 変更 |
| 参加表明カードの項目（作品/キャラ/衣装/時間帯/機材/接近歓迎/撮影OK） | **データモデルに完全に存在**。`EntryCard` を流用 | 流用 |
| 開催予定のイベント（写真＋日付バッジ） | 写真なし。日付バッジ＋テキストカード | 変更 |
| 参加人数ランキング（人物・アバター） | イベント別のまま。アバターなし | 現状維持 |
| モバイル下部タブバー | 追加 | 新規 |

## 実装範囲（MVP）

### 1. ヒーローセクション（`src/app/page.tsx`）

- グラデーション背景（`from-pink-100 via-purple-50 to-violet-50`）を踏襲し、余白・タイポグラフィを強化。
- 大見出し「次のイベント、誰が来るか先に見える。」をグラデーションテキストで大きく。
- サブコピー＋2つのCTA（「イベントを探す」＝ピンク塗り／「参加表明する」＝ブルー塗り or アウトライン）。
- イラストの代わりに、右側（PCのみ）へ**吹き出しカード＋グラデ帯**の装飾ブロックを置く。
  - 吹き出し：「イベント前に、誰に会えるか・何を撮れるかが見える場所」＋アイコン（📷 ♥ ✓）。
  - グラデ帯：「好きでつながる、コスプレ参加表明サイト」。
  - モバイルでは吹き出し・帯は非表示 or 縦積み。
- **統計カード2枚**（開催予定 `{upcomingAll.length}` 件 / 参加表明 `{totalEntries}` 件）を白カードで並べる。

### 2. 検索・絞り込みバー（新規 or `/search` から抽出）

- ヒーロー直下に横並びの検索フォーム＋セレクト（役割／イベント日／エリア／作品ジャンル／絞り込み）。
- 送信で `/search`（または `/events`）へクエリ付き遷移。トップでは入力UIのみ提供し、結果は既存ページに委譲する方針で工数を抑える。
- 既存 `src/app/search/page.tsx` のフィルタ実装を参照し、コンポーネント化を検討（`SearchFilterBar`）。

### 3. 新着の参加表明（横スクロールカルーセル）

- 現状の `grid sm:grid-cols-2` を、横スクロール（`overflow-x-auto` + `snap-x`）のカルーセルに変更。
- カード本体は既存 `EntryCard` を流用（項目はデータモデルに揃っている）。カード幅を固定（例 `w-72`）。
- 「すべて見る →」リンクを見出し右に配置。

### 4. 開催予定のイベント（写真なしカード）

- 日付バッジ（`6.15(日)` のような）＋イベント名＋会場＋参加人数のカード。
- 写真は使わず、日付バッジをカラーブロックで見せる。既存 `EventCard` の刷新 or トップ用の簡易カード。
- 参加人数は `entryCounts[event.id]` を利用。

### 5. 参加人数ランキング（現状維持）

- 現在のイベント別テーブルを踏襲。順位バッジ（金/銀/銅）も現状のまま。アバターは追加しない。
- 見た目の余白・角丸のみ新デザインに合わせて微調整。

### 6. ヘッダー（`src/components/Header.tsx`）

- ナビを整理：「イベントを探す」(`/events`)／「参加表明を探す」(`/search`)／「ランキング」／「使い方」／「お知らせ」。
- **未作成ページの扱いを要決定**：
  - 「ランキング」→ トップの該当セクションへのアンカー or 専用ページ新設。
  - 「使い方」「お知らせ」→ (a) ページを新規作成、(b) 当面ナビから除外、(c) プレースホルダページ。
  - MVPでは既存導線（イベントを探す／参加表明を探す）を優先し、未作成ページはナビから外すか後続対応とする。
- ログインボタン（Xでログイン, 黒）は現状踏襲。

### 7. モバイル下部タブバー（新規）

- モバイル幅のみ表示する固定ボトムナビ（ホーム／探す／参加表明／ランキング／マイページ）。
- `fixed bottom-0` + `sm:hidden`。`main` に下部パディングを追加してコンテンツの被りを防ぐ。
- 新規コンポーネント `MobileTabBar`（`layout.tsx` に配置）。

## 対象外（今回やらないこと）

- ヒーローイラスト等のブランド画像素材の用意・組み込み。
- `Event` への画像フィールド追加、イベントカバー画像の投入。
- 人物別ランキングへの仕様変更（データ集計・アバター表示）。
- 検索結果そのものをトップに表示する機能（入力UIのみ提供し既存ページへ委譲）。

## 影響範囲（変更が見込まれるファイル）

- `src/app/page.tsx` … ヒーロー／統計カード／検索バー／新着カルーセル／イベントカード／ランキング
- `src/components/Header.tsx` … ナビ拡張
- `src/app/layout.tsx` … モバイルタブバー配置、`main` パディング調整
- `src/components/EventCard.tsx`（または新規トップ用カード）… 日付バッジ付きカード
- 新規：`SearchFilterBar`, `MobileTabBar`（必要に応じて）
- `EntryCard.tsx` は流用（変更最小）

## 想定リスク・確認事項

1. **未作成ページ（使い方／お知らせ／ランキング専用ページ）**の扱い。ナビに載せるなら中身が要る。
2. 横スクロールカルーセルのアクセシビリティ（キーボード操作・スクロールヒント）。
3. 検索バーの項目（エリア／作品ジャンルの選択肢）が既存データと整合するか（`region` 等の値域確認）。
4. モバイルタブバーと既存フッターの重なり・余白調整。

## 段階的な進め方（提案）

1. **フェーズ1**：ヒーロー＋統計カード＋新着カルーセル（見た目のインパクトが大きく、依存が少ない）。
2. **フェーズ2**：イベントカード刷新＋ランキング微調整＋検索バー。
3. **フェーズ3**：ヘッダー拡張＋モバイルタブバー（未作成ページの方針決定後）。
